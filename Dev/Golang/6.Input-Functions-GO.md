# Go – Input (Entrées utilisateur)

Ce chapitre fait suite à **Go Output Functions** et se base sur la section **Go Input** de la documentation **W3Schools**. Il explique comment **lire des données saisies par l’utilisateur** depuis la console à l’aide du package `fmt`.

---

## 1. Principe de l’entrée utilisateur en Go

En Go, la lecture des entrées utilisateur se fait principalement avec trois fonctions du package `fmt` :

- `fmt.Scan()`
- `fmt.Scanln()`
- `fmt.Scanf()`

Ces fonctions lisent les données depuis l’**entrée standard** (le clavier).

---

## 2. La fonction `fmt.Scan()`

### Principe

`fmt.Scan()` lit les valeurs saisies par l’utilisateur et les stocke dans des variables.

- La lecture s’arrête à un **espace**, une **tabulation** ou un **retour à la ligne**
- Elle nécessite des **pointeurs** vers les variables

---

### Exemple simple

```go
package main
import "fmt"

func main() {
    var nom string

    fmt.Print("Entrez votre nom : ")
    fmt.Scan(&nom)

    fmt.Println("Bonjour", nom)
}
```

Le symbole `&` permet de passer l’adresse mémoire de la variable.

---

### Lire plusieurs valeurs

```go
var prenom string
var age int

fmt.Scan(&prenom, &age)
```

Entrée utilisateur :

```
Alice 25
```

---

## 3. La fonction `fmt.Scanln()`

### Principe

`fmt.Scanln()` fonctionne comme `Scan()` mais :

- lit jusqu’à la **fin de la ligne**
- provoque une erreur s’il reste des valeurs non lues

---

### Exemple

```go
var ville string

fmt.Print("Entrez votre ville : ")
fmt.Scanln(&ville)
```

`Scanln()` est utile lorsque l’on veut lire **une seule ligne complète**.

---

## 4. La fonction `fmt.Scanf()`

### Principe

`fmt.Scanf()` permet de lire une entrée utilisateur en utilisant un **format précis**, similaire à `Printf()`.

---

### Exemple avec format

```go
var nom string
var age int

fmt.Print("Entrez votre nom et votre âge : ")
fmt.Scanf("%s %d", &nom, &age)
```

Entrée utilisateur :

```
Alice 25
```

---

## 5. Comparaison des fonctions d’entrée

| Fonction | Séparateur       | Cas d’utilisation   |
|----------|------------------|---------------------|
| `Scan`   | espaces / lignes | Entrées simples     |
| `Scanln` | fin de ligne     | Une ligne unique    |
| `Scanf`  | format défini    | Entrées structurées |

---

## 6. Gestion des erreurs

Les fonctions `Scan`, `Scanln` et `Scanf` retournent :

- le nombre de valeurs lues
- une erreur (`error`)

### Exemple

```go
var age int

_, err := fmt.Scan(&age)
if err != nil {
    fmt.Println("Erreur de saisie")
}
```

Toujours vérifier les erreurs dans un programme réel.

---

## 7. Bonnes pratiques

- Toujours afficher un message avant une saisie
- Vérifier les erreurs de lecture
- Utiliser `Scanf()` pour des entrées strictes
- Éviter `Scanln()` pour des saisies complexes
